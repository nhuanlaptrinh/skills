# Program Reference

## Program

- Name: Vibe Coding Đến Tự Động Hóa Công Việc Với Python.
- Brand/teacher: Anh Lập Trình, teacher Nhuần.
- Main positioning: Help working adults and complete beginners create apps/tools for their own work by starting with AI-assisted vibe coding, then learning Python deeply enough to customize, fix, and build flexible automation projects.
- Core method: "Thực hành trước - lý thuyết sau - vướng đâu, gỡ đó."
- Secondary slogan: "Cứ ứng dụng vào công việc đi, vướng thì gỡ."

## Core Promise

- Beginners can start without computer science or programming background.
- Learners begin by creating something useful quickly so they feel the practical value first.
- The course is not abstract Python theory. It is centered on real work problems and the learner's own project.
- The learner should move from repetitive manual work to custom tools/apps that serve their actual job.

## Two-Stage Learning Path

### Stage 1: Vibe Coding With Antigravity AI

- Use Antigravity AI to create web apps and tools by giving instructions.
- The learner does not need to understand code at the beginning.
- Antigravity AI helps with setup, testing, error checking, and tool creation.
- Goal: create a useful tool fast enough that the learner feels excited and wants to continue.
- Example: tell Antigravity AI to create a customer management app with Excel, then review and adjust details.

### Stage 2: Python Automation

- After creating tools with AI, learn Python to understand how the app works.
- Use Python to customize, debug, and build more flexible or complex projects.
- Topics/examples:
  - Automate Excel, PDF, Word, and websites.
  - Collect and summarize data from websites such as ecommerce, social, invoice, and reporting portals.
  - Send automated emails with attachments and schedules.
  - Control mouse and keyboard.
  - Use regular expressions to recognize/search text.
  - Use Antigravity AI, Google, VSCode, and other tools to find/fix errors.
  - Automate repetitive computer tasks end to end.

## Audience

- Office workers tired of repetitive manual tasks.
- People in accounting, HR, sales, technical, operations, admin, and similar work.
- People who want extra income from automation, outsourcing, small software, or consulting.
- People considering a cross-border tech path or career change.
- People who tried Python before but quit because lessons were too theoretical and not connected to real work.
- People over 30 or around 40 who have practical work experience and clear problems to solve.

## Beliefs To Challenge

- "Programming requires strong English."
- "Programming requires strong math."
- "Programming is only for very smart people."
- "Older people cannot learn programming."
- "You must learn theory first before building anything useful."
- "Ready-made software will always fit your work."

## Teacher Story/Positioning

- Nhuần presents himself as a practical, self-taught, non-elite teacher from a technical/mechanical construction background.
- He says he was not strong in English or math, started programming around age 30, and learned fast because he had pressure, work experience, and concrete project ideas.
- He positions himself against "born excellent" teachers by saying he understands beginners and working adults because he came from the same side.
- He frames learning programming like learning to drive: let learners move first, feel capable, then learn deeper controls and theory.

## Learning Format And Support

- Learn by video first, so busy working adults can study whenever they have time.
- Apply immediately to the learner's own job.
- Get help through Zalo, Zoom, video, and UltraViewer when stuck.
- Course content updates continuously based on learner needs.
- The learning style can feel difficult at the beginning because programming is unfamiliar; seriousness and persistence matter.

## Common Story Arcs

### Repetitive Work To Custom Tool

The reader spends days copying, pasting, cleaning Excel files, downloading reports, sending emails, or checking websites. They look for existing software, but nothing fits because only they understand the messy reality of their job. The course offers a path to build their own tool: first fast with AI, then flexible with Python.

### Failed Python Learning To Practical Learning

The reader tried a Python course, learned variables/loops/functions, got bored, and could not connect lessons to work. This course flips the order: build something useful first, then learn the Python needed to improve it.

### Late Starter To Advantage

The reader thinks they are too old or not smart enough. Reframe their work experience as an advantage: older learners have real problems, clearer goals, and more ideas for automation.

### Career Visibility And Extra Income

The reader creates small automation tools for their own team, gets noticed by colleagues or managers, then may expand into side projects or outsourcing. Keep this as a possible path, not a guarantee.
